---

### Phase 1: Recommended Technology Choices

To use Copilot, Vercel, and Neon effectively, here is the recommended tech stack:

* **Database:** Neon (PostgreSQL) is perfect. PostgreSQL's robustness is needed for complex data and relationship management.
* **Framework:** **Next.js** (App Router). Copilot has immense knowledge of Next.js. Vercel is the creator of Next.js, making deployment seamless.
* **Language:** TypeScript. This is crucial for managing the complex objects (like dynamic assessment structures) you’ll be handling. Copilot will write excellent TypeScript.
* **ORM (Object-Relational Mapping):** **Prisma** or **Drizzle**. These tools map your code to your Neon database. They are Copilot’s "native language" for database interactions.
* **Styling:** Tailwind CSS (for the UI) and a library like `react-hook-form` and `zod` for handling dynamic forms.
* **PDF Generation:** `react-pdf` (client-side) or `puppeteer-core` with Vercel functions (server-side, more robust).

### Phase 2: Architecting the Database (Neon)

This is the most critical step. Your database schema must handle dynamic questionnaires and scoring logic.

Here is a high-level conceptual schema that you can give GitHub Copilot as a starting prompt. It shows how assessments are defined *vs.* how customer submissions are stored.

```typescript
// Define what an Assessment Template looks like
model AssessmentDefinition {
  id                    String    @id @default(cuid())
  title                 String    // e.g., "Security Audit"
  description           String?
  // Stores the entire list of questions as a JSON object
  questions             Json      // Example structure: [{id: 1, text: "X?", type: "text"}, {id: 2, text: "Y?", type: "dropdown", options: ["A", "B"]}]
  // Stores the logic for calculating scores based on answers
  scoringLogic          Json      // Define point values for specific answers
  // The knowledge base used to generate recommendations
  knowledgeBase          Json      
  submissions           CustomerSubmission[]
}

// Define what a customer's specific submission looks like
model CustomerSubmission {
  id                    String              @id @default(cuid())
  customerName          String
  customerEmail         String?             // For sending the link
  status                SubmissionStatus    // IN_PROGRESS, COMPLETED, DELIVERED
  uniqueLinkToken       String?             @unique // The magic link for the customer
  assessmentDefinitionId String
  assessmentDefinition  AssessmentDefinition @relation(fields: [assessmentDefinitionId], references: [id])
  // Stores the customer's specific responses as JSON
  answers               Json                // Example structure: {question_id_1: "Answer text", question_id_2: "A"}
  finalScore            Float?
  recommendationReport  Json?               // The generated recommendation data
  pdfUrl                String?             // Once generated, store the Vercel Blob URL here
  createdAt             DateTime            @default(now())
  updatedAt             DateTime            @updatedAt
}

enum SubmissionStatus {
  IN_PROGRESS
  COMPLETED
  DELIVERED
}

// Basic User model for portal login
model User {
  id                    String    @id @default(cuid())
  name                  String
  email                 String    @unique
  role                  String    @default("editor") // You can simplify as everyone has "full access"
}

```

### Phase 3: Copilot Implementation Guide

Now you can start building the features in VS Code. Use Copilot to tackle one component at a time.

#### 1. Designing Dynamic Assessments (The Admin View)

This is your primary challenge: creating an interface that lets you dynamically build forms.

* **Copilot Prompt 1:** "Using React, React-Hook-Form, and Tailwind, build a component `AssessmentBuilder` that lets me add, reorder, and edit questions. For each question, I should be able to select an input type (text, textarea, dropdown, radio button, checkbox) and define option values."
* **Copilot Prompt 2:** "Create a schema for storing 'scoring logic' in a JSON field. If a dropdown question has 'Yes' or 'No', 'Yes' should be 10 points and 'No' should be 0 points. Build a UI component to input this logic for each question."

#### 2. Assessment Launch & Link Generation (The Portal View)

You need to handle two starting states: portal user input *vs.* magic link.

* **Scenario A: Capture Customer Name.** You load the assessment definition in the portal and save a new `CustomerSubmission` row immediately.
* **Scenario B: Send Assessment Link.** Copilot can help you generate a secure, unique URL.
* **Copilot Prompt:** "I need a backend API route in Next.js that takes an `AssessmentDefinitionId` and a customer email, creates a new `CustomerSubmission` entry with a status of 'IN_PROGRESS' and a randomly generated unique token, and then uses Resend/SendGrid to email the link `/assess/[uniqueToken]` to the customer."



#### 3. Scoring & Recommendations Logic (The Core)

This is the heart of the application, transforming raw data into intelligence.

* **How to structure it:** The JSON `answers` are compared against the JSON `scoringLogic`. Copilot can write the JavaScript function to perform this calculation.
* **Recommendations:** You need a function that maps scores or specific "bad" answers to a section from your "recommendation knowledge base."
* **Copilot Prompt:** "Write a TypeScript function `calculateScoreAndRecommendation(answers, scoringLogic, knowledgeBase)`. It will iterate through the customer's answers, look up the point value in `scoringLogic`, calculate the total score, and generate a recommended action list based on which scores fell below a threshold defined in the `knowledgeBase`."

#### 4. The Customer Assessment Page (The Public View)

This page is shown when the customer clicks the email link. It is unauthenticated (only controlled by the unique token).

* **Copilot Prompt:** "Build a Next.js public page `/assess/[token]`. It should query the database for a `CustomerSubmission` by token. It then dynamically renders the assessment form based on the questions defined in the associated `AssessmentDefinition`. When the customer submits the form, it updates the `CustomerSubmission` with their answers and sets the status to 'COMPLETED'."

#### 5. PDF Generation (The Final Deliverable)

Generating the PDF requires taking dynamic HTML and converting it into a static document.

* **Technique:** Vercel (your host) has some great native tools for this. `@vercel/og` can be adapted for image generation, but for robust PDFs, the standard is often running Puppeteer in a Vercel function.
* **Copilot Prompt:** "I need a serverless function in Next.js that takes the generated recommendations for a specific assessment, loads an HTML template, and uses `puppeteer-core` and `@sparticuz/chromium-min` to generate a PDF. The output PDF should be uploaded to Vercel Blob storage, and the resulting URL should be saved in the database under `pdfUrl`."
